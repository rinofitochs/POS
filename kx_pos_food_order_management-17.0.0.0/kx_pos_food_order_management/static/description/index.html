Overview

The kitchen screen module in Odoo 17 streamlines kitchen operations by facilitating efficient order management and enhanced communication. This module swiftly transmits orders to the kitchen, enabling cooks to view and manage their tasks effectively. It provides real-time order updates, ensuring that kitchen staff are always informed of current orders and their status. Chef have the flexibility to customize the information displayed, allowing them to focus on relevant details and optimize their workflow. By integrating these features, the kitchen screen module significantly boosts efficiency, supports a seamless kitchen operation, and enhances overall productivity in the culinary environment.

Generate kitchen screen interface from POS

For smooth order processing, ensure your POS session and connected categories are set up to automatically send orders
to the dedicated kitchen screen. To view detailed information for these orders, simply click the 'Kitchen Screen' button highlighted
in the image above.

Essentially, the kitchen screen starts with a clean slate until new orders are generated and subsequently displayed.

Simplify order creation with the dedicated 'Order' button.

Upon order creation within the POS session, the order transitions to the 'Cooking' stage. At this stage, detailed information is available
for each order, including the Order Time, Table Name, Cashier Name, Order Internal Reference, and Floor Details. Additionally,
the total number of orders in the 'Cooking' stage is tracked, providing a comprehensive overview of kitchen operations.

After accepting the order in the 'Cooking' stage, it transitions to the 'Ready' stage. Once the food is prepared, clicking the 'DONE' button
will automatically move the order to the 'Completed' stage. During this process, all order details remain accessible for review.

All completed orders are displayed in the 'Completed' stage. This stage provides a detailed view of each order,
including essential information such as Order Time, Table Name, Cashier Name, Order Internal Reference,
and Floor Details, ensuring comprehensive oversight.

Payment processing is activated only after an order reaches the 'Completed' stage, enforcing a structured workflow that supports
efficient kitchen operations. Any attempt to process payment before completing all kitchen orders triggers an error, ensuring
that all food items are prepared prior to finalizing transactions.

Upon order finalization, the system enforces order immutability by preventing subsequent item additions, thereby ensuring
data integrity and operational efficiency.

Facilitate the creation of additional orders for the same table by selecting the 'Order' button positioned in the top-right corner.
